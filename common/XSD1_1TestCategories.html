    1. maxOccurs may now be >1
    2. wildcards are now allowed
    3. derivation by extension
    4. derivation by restriction
    5. named model group references in xs:all
  2. Assertions
    1. on complex types
    2. on simple types
    3. derivation by restriction (assertions of base type inherited)
    4. derivation by extension (assertions of base type inherited)
    5. rules for whether the XDM view is typed or untyped
    6. rules for the static and dynamic context of the XPath expression
    7. inability to navigate outside the subtree
    8. errors in XPath evaluation are treated as a false result
    9. impact of xpathDefaultNamespace locally and at schema level
  3. Conditional Type Assignment
    1. basic
    2. xs:error type
    3. derivation by restriction
    4. inheritable attributes
    5. XPath expression can only access attributes
    6. static and dynamic context of XPath expression
    7. static errors in XPath expression
    8. dynamic errors in evaluation of XPath expression
    9. all alternatives must be substitutable for the declared type
    10. default branch
    11. impact of xpathDefaultNamespace locally and at schema level
  4. Open content
    1. suffix open content
    2. interleave open content
    3. with element-only content model
    4. with mixed content model
    5. derivation by extension
    6. derivation by restriction, including mode="none"
    7. with complex type with simple content (ineffective)
    8. <openContent> id attribute
    9. <openContent> annotation child

    11. defaultOpenContent different across schema documents
    12. defaultOpenContent appliesToEmpty = true
    13. defaultOpenContent appliesToEmpty = false
    14. defaultOpenContent applies to types within xs:redefine
    15. defaultOpenContent applies to types within xs:override
  5. Substitution groups
    1. Element in more than one subst group
    2. Abstract elements allowed in subst group
  6. New features for local declarations
    1. Target namespace on local element declarations
    2. Target namespace on local attribute declarations
    3. Named identity constraints on local element declarations
  7. vc: conditional inclusion
    1. Testing version
    2. Testing supported types
    3. Testing supported facets
    4. Unrecognized attributes in vc: namespace
  8. Wildcards
    1. not namespace
    2. not QName
    3. not #defined
    4. not #definedSibling
    5. relaxation of UPA in the presence of wildcards
    6. skip wildcards now excluded from EDC constraint
    7. tighter rule for EDC as regards the type of an element that matches a wildcard
  9. Date/time types
    1. ExplicitTimezone facet
    2. dateTimeStamp derived type
    3. Allow year zero
    4. Disallow leap seconds
    5. Equality of (for example) PT1D and PT24H
    6. Built in derived types xs:yearMonthDuration and xs:dayTimeDuration
  10. PrecisionDecimal
    1. Validation of precisionDecimal values
    2. precisionDecimal facets
  11. Union types
    1. Types derived by restriction from a union type
    2. xsi:type used to name a member of a restricted union type
    3. Cyclicity of union types
  12. XML 1.1 support
    1. Value space of xs:Name and related types
    2. Support for additional Unicode characters in content
    3. Use of newly allowed name characters in names of schema components (e.g. types, model groups)
    4. Effect of \i and \c in regular expressions
  13. Restriction of complex types
    1. "Intensional restriction": allows some restrictions that 1.0 disallowed
    2. Restrict xs:all to xs:sequence
    3. Restrict xs:all to xs:choice
  14. Miscellaneous
    1. Default/fixed values for xsi:type and xsi:nil
    2. xsi:type fallback to lax validation when
    3. xsi:type must resolve to a type definition
    4. xs:anyAtomicType
    5. complexType/@mixed and complexContent/@mixed must not conflict
    6. equality of an atomic value with a singleton list in ID/IDREF constraints
    7. impact of xpathDefaultNamespace locally and globally on key/keyref/unique selection expressions
    8. lexical representation +INF for float and double
    9. use of hyphens within square brackets in regular expressions
  15. Schema composition
    1. xs:override
    2. Referring to a namespace requires an xs:import (probably already in 1.0 suite)
    3. ... except for the xs and xsi namespaces
    4. Cyclic dependencies redefine-include-import-override

  16. Default attribute group
    1. with defaultAttributesApply=true or defaulted
    2. with defaultAttributesApply=false
    3. different across multiple schema documents
    4. applies to types within xs:redefine
    5. applies to types within xs:override
  17. ID/IDREF
    1. Lists of ID
    2. Unions involving ID
    3. Multiple attributes of type ID
    4. Default values of type IDREF, ENTITY, or ENTITIES
    5. Default values for elements or attributes of type ID
